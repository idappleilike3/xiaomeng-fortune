## 0 設計理念(對齊 3-Layer UX v1.0 §1)

### 核心原則:**漸進式揭露(Progressive Disclosure)**

| 舊 | v1.0 |
|---|---|
| 首頁一次展示 20-30 個功能 | 首頁只顯示 **4 個入口** |
| 新使用者選擇困難 | 一次只回答一個問題 |
| 直接展示所有牌陣 | 牌陣隱藏到第 3 層 |
| 「我要抽哪一種牌陣?」 | 「**今天想探索什麼?**」 |

### UX 四步原則(老闆硬規則)

```
第一步:選系統
第二步:選主題
第三步:選牌陣
第四步:開始神聖儀式
```

**未來規則:** 任何新增牌陣 → 永遠 L3,首頁不變。

## 2 網站架構(對齊 HOMEPAGE_3LAYER_UX v1.0 §2)

### 2.1 4 層架構

```
L1 首頁(4 入口 — 永遠簡潔):
├─ 🔮 命運塔羅
├─ 🐾 毛孩心語
├─ 🌙 今日神諭
└─ ✨ 神殿深處(探索全部)

L2 主題選擇(進 L1 任一入口後):
├─ 命運塔羅 → 熱門 / 感情 / 工作 / 財富 / 自我探索 / 更多
├─ 毛孩心語 → 心情 / 健康 / 關係 / 更多
└─ 今日神諭 → 今日訊息 / 宇宙提醒 / 心靈祝福(無「更多」)

L3 神殿深處(進 L2 任一主題後):
├─ 🌌 星盤解讀
├─ 🔢 生命靈數
├─ 🪐 流年運勢
├─ 🧭 命盤諮詢
├─ 💌 深夜靈性來信
├─ 🎁 命運市集
├─ 👑 會員中心
└─ ⚙ 個人設定

L4 F22 神聖舞台(進 L3 任一牌陣後):
└─ 11 步編舞 → 完成
```

### 2.2 首頁永遠簡潔的硬規則

| 規則 | 違反 = reject |
|---|---|
| 首頁只 4 入口 | 新增第 5 入口需 ARCHITECTURE PR |
| 首頁不展示牌陣 | 任何牌陣卡片在首頁 = reject |
| 首頁不展開 F30 細項 | 直接列具體牌陣 = reject |
| 首頁不超過 1 屏 fold | 超過 = reject |
| 任何新增牌陣 → 永遠 L3 | 不在首頁 / 不在 L1 |

## 3 每日回來機制(Storyboard v3.0 §11 + SP Ecosystem v1.0 §5)

```
使用者每天回來時,首頁出現:

🌞 今日儀式

☐ 今日神諭 (+5 SP)
☐ 今日免費探索
```
